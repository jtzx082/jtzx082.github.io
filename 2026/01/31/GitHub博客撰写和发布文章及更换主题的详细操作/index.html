第一部分：如何撰写并发布文章（日常操作）

Hexo 的文章都是 Markdown 格式的文件。你以后发布文章的流程就像“写日记 -> 预览 -> 发布”一样简单。

1. 新建文章

提示：系统会在 source/_posts/ 目录下生成一个 我的第一篇文章.md 文件。

2. 撰写内容

找到这个文件并编辑它。

  • 推荐工具：在 Ubuntu 上，强烈推荐使用 VS Code 或 Typora（如果能安装的话）来编辑 Markdown。
  • 文件格式：打开文件你会看到顶部有两条虚线，那是“Front-matter”，用来定义标题、时间、标签等。正文写在虚线下方即可。

3. 本地预览（重要）

写完后，先在本地看看效果，防止排版出错。

打开浏览器访问 http://localhost:4000。如果不满意，改完文件保存，刷新浏览器就能看到变化（不需要重启 hexo s）。

给“高手”的进阶建议

  1. 主题的配置：
    每个主题在它自己的文件夹里（例如 themes/next/）都有一个独立的 _config.yml 配置文件。
    你可以修改它来开启侧边栏头像、社交链接（GitHub、Email）、阅读量统计、评论系统等高级功能。建议去搜一下“Hexo Next 主题美化”，教程非常多。
  2. 图片处理：
    这是新手最容易遇到的坑。Markdown 里的图片不能直接粘贴。
    推荐做法：
  • 在 Ubuntu 上安装 VS Code，并安装 Paste Image 插件。
  • 设置插件将图片保存到 source/images 目录。
  • 这样你在写文章时，截图后直接 Ctrl+V，图片就会自动保存并插入文章中。
